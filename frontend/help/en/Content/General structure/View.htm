View

Hover over image to enlarge

A view provides a means of viewing and interacting with data or images via SVOWebApp. It is the basic element of a view set.

You can find more information about view sets under: View Set

If you have added content to a view, it has the following components:

  1. View title
  2. View controls
  3. View content
  4. View type controls

If you have not yet edited a view, it only contains a single, empty view, where you can add a view and which includes only the view controls in the top right corner. The view title as well as the view type controls are shown if you have added content.

View Title

The view title provides more information about the source and the content type.

The view title indicates the inspection and, depending on the view type, the specific inspection tool or the view type of the added view content. If the view is empty, no title is shown.

View Controls

You can add view content via the general view controls located in the upper right corner of the view.

Views include the following general controls:

Element Description

Split window

Choose view type
Delete view
Add view to view set

You can add as many views to a view set as you desire. The views are initially empty.

→ In the upper right corner of the view, click on Split window .

→ The view is split either on a horizontal or vertical dividing line, depending on the original shape of the view to be split.

View Content

The actual content of the view may include data, parameters, values or images which are visualized within the view. The displayed view content depends on the inspection and the view type (e.g. image, parameter, image strip, etc.) that you selected.

 

You can find more information about view types under: View types

Add view content

You can add view content, such as data, parameters or images to a view by selecting a specific inspection and choosing from a list of predefined view types.

→ Click on Choose view type to display the following screen where you can select a view type. This step is not necessary when you add a view to a view set or when you set up a view set from scratch.

View Type Controls
