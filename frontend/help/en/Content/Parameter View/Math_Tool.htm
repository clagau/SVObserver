Math Tool

General

The Math tool enables you to perform mathematical and/or logical operations, using specified formulas, on results data from any of the tools in the same tool set as the Math tool. Your application can use the results data from these Math tool operations, in addition to results data from individual tools to control your application. Results from individual tools can be combined to provide a single result for an output or another tool.

Adding of the Math Tool to a View Set

The Math Tool is part of the Tool Set View. The tool's formula, ranges, condition and comment can be edited in a Parameter View.

→ Add a Tool Set View for an inspection.

→ Highlight a tool and add a Math Tool, if there is not yet one that originates from the SVObserver configuration.

→ The Math Tool is added after the tool you highlighted.

→ Split the window and add a Parameter View for the same inspection.

→ The Tool Set View and the Parameter View are now connected.

→ You can check the connection by selecting one of the sub-nodes "Formula", "Range", "Conditional" or "Comment". The Parameter View changes its content.

 

Formula

A formula can be built using linked values, variables and functions. Formulas use syntax highlighting. You can enter decimal, binary and hexadecimal numbers.

How to insert a formula

→ Highlight the node Formula in the Tool Set View.

→ Click on "Open edit view" next to "Formula".

→ The formula editor opens.

You can now insert a formula by using the buttons for numbers, linked values, operators and mathematical symbols.

How to add a linked value

You can add variables from tool results and PPQ variables.

→ Click on "Add linked value".

→ Navigate to a node by clicking on a triangle to expand the sub-nodes.

→ Click on a linked value on the right to insert it in the formula.

Array Variables

The results from a blob feature array can be used as part of a formula. Both individual element and whole array results can be used in equations.

The size of the array or number of elements reported for each feature is set by the Maximum Blob Data Array Size setting. There is an array for each feature in the Current Features table. The blobs in that array are sorted using the Sort Feature criteria.

 

Single Array Elements

Single array element results are reported as: “Tool Set name.Tool name.Feature type” [i] where “i” is the element position. An example would be “Tool Set.W_blob Tool.Area” [3], where the area of the first blob would be the result.

By default, a feature result without an element “[i]” returns the result for the first element [1].

If the number of blobs that are found with Blob Analyzer is less than the Maximum blob data array size, the default value is reported as the result. The default value can be set in the Set Feature Properties dialog. The default value is set for each feature. The default values may be different for different features within a blob analyzer. The default value may also be set in the result when used in a Math tool. If set within the Math tool, the setting in the Feature Properties is overridden.

The default value is entered into the formula by entering a “{default value}” after the element and before the closing bracket “ ] ”. In the example shown below, if there was not a ninth blob found, the element will report the default value of –999.

If the number of blobs that are found with Blob Analyzer 2 is less than the Maximum blob data array size, an invalid value is reported as result for blobs exceeding the number of blobs that are found.

 

Whole Array Results

Whole array results are reported as: “Tool Set name.Tool name.Feature type” [ ] where there is no number between the brackets.

Example:

“Tool Set.Window Tool.Area” [ ] , where the area of all blobs would be the available as the result.

This functionality can be used in conjunction with the MIN, MAX, AVG, MED, SUM and STD DEV functions. The function uses all valid elements from the feature array for the calculation. Any elements that did not have a blob will be disregarded.

Multiple blob arrays can be used together in the above-mentioned function. The arrays should be separated using a comma. The elements will be all treated equally in the function. If the following function was created: AVG (“Blob_A” [ ], (“Blob_B”), where Blob_A and Blob_B had 100 and 15 elements respectively, the AVG function would compute the result based on 115 values.

 

Blob Array Example

As an example of how to use the equation array capabilities, we will compute the difference in size between the first blob (sorted by area) and the average area for all blob results.

Select the Variables browse button and select Area[1] of a Blob analyzer. Click to the end of the line and type the minus sign. Click on the AVG button, delete the comma, and select Area[ ] of a Blob analyzer as a linked value. Click on Validate return the result of the equation.

 

Range

Ranges provide a quantitative measure against which the results from each inspection tool are compared. There are “fail” and “warning” limits, each has an upper, and lower limit that you can set as is appropriate for the particular inspection tool. The range of allowable values is -17,000,000 to 17,000,000.

 

The software compares the inspection results value to the range limits specified for that tool. The tool status is a function of where an inspection results value lies in relation to the range limits.

 

Fail, Warn, Pass

The Pass, Fail, and Warn statuses are mutually exclusive.

The order of precedence is 1) Fail, 2) Warn, 3) Pass.

Fail

  • If the Result Value is greater than the Fail High value, the evaluated result is marked as Fail.
  • If the Result Value is less than the Fail Low value, the evaluated result is marked as Fail.

Warn

Fail takes precedence over warn. If a Fail has already been determined to be true, there is no test for Warn.

  • If the Result Value is greater than the Warn High value, the evaluated result is marked as Warn.
  • If the Result Value is less than the Warn Low value, the evaluated result is marked as Warn.

Pass

An evaluated result receives Pass when it does not fail and warn.

 

How to enter a range
  1. Click on the "Open edit view" button next to a limit for fail or warn.
  2. Select a type.

    If you choose "Value":

    • Enter a direct value in the field below.
    • Use the number buttons to enter a value.
    • Use the arrow buttons to increase or decrease the value by 1 or by 10.

    If you choose "Link":

    • Click on the "Open edit view" button below.
    • Select a node in the tree view and a value on the right.
    • Confirm and close the dialog by clicking on OK.

    If you choose "Formula"

    • Click on the "Open edit view" button below.
    • Enter your formula and validate it.
    • Close the dialog by clicking on OK.
  3. Close the dialog by clicking on OK.
  4. If you entered a formula, it is displayed next to the range.
  5. If you selected a linked value, it is displayed next to the range.
  6. Direct values are not highlighted. They are displayed as simple number.

 

Type of values

You can define a direct value, a linked value or a formula that is used as value for each range.

 

Detailed information about linked values

A linked value is a reference to the value of another output.

Any integer, real, or Boolean result that are normally accessible within the given inspection can be selected. The field is validated before closing the dialog. An error message is displayed if the entered name is not a valid and accessible result item. The system does not verify that the referenced result values follow the order rules that are used for constant values.

Ensure that the referenced range values allow for a correctly ordered pass/warn/fail range. If the pass/warn/fail range are not ordered correctly (ex. Fail Low higher than Fail High), the tested result value is always either higher than the Fail High or Lower than the Fail Low. In these cases, the result (and usually the tool) fails. The rules of precedence (Fail/Warn/Pass) still apply.

Please note that forward referencing an inspection result results in getting the value from the previous iteration of the inspection. It should also be understood that the order of execution counts. If a current tool result that has not yet been calculated is referenced, the value used is the value from the previous inspection. Do not assume that all results are calculated prior to evaluating the ranges regardless if any of the results are within range.

Conditional

The operation of the Math Tool can be made based on certain conditions being fulfilled. The tool is executed if the result of the formula is a TRUE or non-zero value. If the result of the formula is zero, the tool is not executed.

 

You can create the condition formula by selecting values, variables and operators and inserting in formula box.

 

See above Formula for more information about how to enter a formula.

 

Comment

In the Comments node for the Math Tool, you can add and edit comments about the tool or the tool set in the field. Comments can be used to provide information about the settings of the tool or the tool set.
