Connection

SVOWebApp can connect to the inspection system via IP address or the computer name and port 80.

The connection to the inspection system must run via the protocol SVOGateway. It allows for the communication between SVObserver and clients, such as machine HMI and Internet browser.

When the configuration that you want to use was created with a previous version of SVObserver that is not matching the versions of SVObserver and SVOWebApp, the following error message shows: "Error while configuration: Error: Selected config is not compatible with the current WebApp version."

How to solve this configuration issue
Note

The currently only solution involves that the SVOWebApp configuration is deleted. This means that you will lose any configured view sets and views.

→ Go to SVObserver.

→ Open the folder "C:\RUN" in Explorer.

→ Delete or rename the file "WebApp.json". It contains the SVOWebApp configuration.

→ Refresh the SVOWebApp interface.

→ Configure SVOWebApp by adding views.

→ Save the view set in the View Manager.

→ The file "WebApp.json" is created again.

Note

SVObserver is a high-performance inspection system. Connections to other networks than to the internal machine network are permitted. It is not allowed to install additional applications or services onto the inspection system. Virus scanners or firewalls are not allowed to run on the inspection system. SVRC Seidenader Vision Remote Control clients need to be compatible to the running version of SVObserver.
