View

A view provides a means of viewing and interacting with data or images via the SVOWebApp. It is the basic element of a view set.

For more information on view sets, read topic: View set 

A view with added content will have the following components:

  1. View title
  2. View controls
  3. View object
  4. View context controls

A view set which has not yet been edited will only contain a single, empty view, which includes only the view controls in the top right corner. The view title as well as the view context controls will be indicated once a view object has been added.

 

View Title

Once an empty view has been edited, the view title indicates the group and view type of the added view content.

The view title provides more information about the source and the content type

 

View Controls

The user can add view content via the general view controls located in the upper right corner of the view

The view includes following general controls:

Element Description
Add view object, such as data, parameters, images
Enlarge view

Split view  
Delete view
Add view to view set

You can add as many views to a view set as you desire. The views will be initially empty.

  • In the upper right corner of the view, click on Split view

    — The view will be split either on a horizontal or vertical dividing line, depending on the original shape of the view to be split

 

View Object

The view object is the actual content of the view and may include data, parameters or images which are featured within the view. The indicated view objects depend on the group (e.g. inspection) and view type (e.g. image, parameter, image strip, etc.) of the added view content.

 

For more information on view types, read topic: View types  

 

Add view object

You can add view content, such as data, parameters or images to a view by selecting a group (e.g. a specific inspection) and choosing from a list of predefined view types.

For more information on view types, read topic: View types 

  1. In the upper right corner of the view, click on Add view object

    — Submenu opens. This submenu indicates the available groups, e.g. inspections

  2. Click on the desired item of the group submenu

    Example: The groups submenu indicates various inspections

    — A second submenu opens. This submenu indicates the available view types

  3. Click on the desired item of the view type submenu

    Example: The view type submenu indicates the available view types

    — The view object will be displayed in the center of the view

 

View Context Controls

Once an empty view has been edited, this context sensitive controls bar will indicate the controls relevant to the selected view type. The view context controls are explained in topics of the respective view types.

 

For more information on view types, read topic: View types
