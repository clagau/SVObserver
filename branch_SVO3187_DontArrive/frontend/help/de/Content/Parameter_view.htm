Parameter View

This view type indicates parameters and their values in reference to a selected image view. The parameter view is context sensitive, as this view always displays the parameters that are applied to a custom overlay in an Image View when in run mode.

For example, a parameter view that is targeted to an instance of image view includes all relevant parameters to manage one or more custom overlays.

Custom Overlay Parameters

Hover over image to enlarge

You can create and configure custom overlays in a parameter view. The target image and the overlay are featured in an image view. You can apply multiple custom overlays to one instance of an image view.

You can find more details on custom overlays and their difference to inspection overlays in Overlays.

The parameter view for custom overlays includes the following specific components:

  1. Overlay list
  2. Overlay parameters
  3. Parameter controls

Overlay List

This list indicates all custom overlays for the target image. Inspection overlays are not listed in the overlay list. You have to add custom overlays by using the Parameter Controls at the bottom of the view.

Note

There might be no entry in the overlay list until you have created a custom overlay.

Hide custom overlays in image view

You have two options to hide custom overlays in the image:

Hide all custom overlays

→ Uncheck the checkbox next to the heading Overlays.

→ Custom overlays are no longer displayed with the image.

Hide individual custom overlays

→ Uncheck the checkbox next to the custom overlay you want to hide .

→ The respective custom overlay is no longer displayed with the image.

Show custom overlays in image view

You have two options to display custom overlays in the featured image:

Show all custom overlays

→ Activate the checkbox next to the heading Overlays.

→ Custom overlays are displayed in the image.

Show individual custom overlays

→ Check the checkbox next to the custom overlay you want to show .

→ The respective custom overlay is displayed in the image.

Note

The overlay list indicates only the custom overlays that refer to the currently selected image view of the same inspection. This means that the custom overlays shown in the list change when you switch between the image views.

Overlay parameters

There are various input fields to add parameters of the custom overlay. You can change the parameters for the selected custom overlay.

Certain fields allow to enter constant values. These values stay the same when in run mode.

You can select variable values in certain fields that have a small "x" at the end. The values of these variables are updated when in run mode depending on the SVObserver configuration.

For more information about how to edit parameters, see Edit parameters for a custom overlay.

The following overlay parameters are available:

Exceeding maximum allowed values

The maximum value for dimensions-related parameters (X, Y, Offset Left, Offset Top, Width and Height) is fixed to two times the image size.

If you enter a fixed value for one of these parameters, the value changes automatically to the maximum allowed value.

If the value for a variable exceeds the allowed maximum, you see in the notifications area a notification specifying the allowed range. If you expand the list of notifications, you see a counter that shows how often the value has already exceeded the maximum. The maximum allowed value is used to visualize the overlay.

Example:

The image has a size of 100 px. The maximum value is 200 px. When the variable entered for X has a value of 250 px, the overlay is positioned at 200 px as it is the maximum value. You see a notification specifying that the value should be within a range of minus 200 to 200 px.

Parameter Controls 

The parameter view offers the following elements to control the parameters (from left to right):

Element Description

Add custom overlay

Delete custom overlay

Note: Clicking this button deletes the custom overlay that is currently selected. The custom overlay is no longer available for future use and the parameter set is permanently erased.
Add custom overlay

You can create several overlays for a target image by adding them to the overlay list in the respective parameter view.

→ In the view menu, click on the button Add overlay.

→ A new custom overlay is added to the overlay list.

Edit parameters for a custom overlay

After you added a custom overlay, you can edit all parameters for this custom overlay. You can choose to enter constant or variable values.

→ Click in the parameter field to enter or change a value.

→ If the cursor is placed inside the field, you can enter a value manually. (Refer to Overlay Parameters for more information and character restrictions.)

→ If a selection tree opens in a pop-up window, navigate through the parameters tree to the parameter you want to select. Click in the box next to the desired parameter. (For more information about how to navigate through the selection tree, see Selection Tree)

→ If a drop-down list opens, select the desired parameter by clicking on it.

All changes are applied immediately on the custom overlay in the associated image view.

The actual parameter value is not indicated in the parameter list. In some cases, it might be useful to view the value referring to the applied parameter. You have to create a view of the type list for the same inspection. You can find more information about how to display the parameter values under Show values of selected parameters in List View.

Delete custom overlay

Besides the option of hiding a custom overlay in the image view, you can also delete a custom overlay.

→ In the overlay list, click on the custom overlay you want to delete.

→ In the parameters control, click on the button Delete.

Note

Clicking this button deletes the custom overlay that is currently selected. The custom overlay is no longer available for future use and the parameter set is permanently erased.

Target Image

The dependency between the custom overlay parameters and the target image featured in an image view is visualized by a green frame around the target image view.

Display custom overlay in image view

To view one or more custom overlays on an image as well as the overlay parameters, following views are required:

  1. One instance of image view that features the target image as well as the custom overlays

  2. One instance of parameter view for the same inspection to edit the parameters for the custom overlay

Both instances of image view and parameter view have to refer to the same inspection as indicated in the view title.

If you add a parameter view for a specific inspection and there is no image view for this inspection in the current view set, the following message appears: "Please select a source for this parameter view."

Associate parameter view and image views

You can establish the dependency between one instance of parameter view and several instances of image view referring to the same inspection by following these steps:

→ Add the instances of image view that you want to display for one inspection.

→ Add one parameter view for the same inspection.

→ Click on the instance of the image view in which you want to add a custom overlay. A green frame is displayed to show that this image view is selected.

→ You can now add a custom overlay and edit its parameters.
