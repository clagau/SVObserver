Permissions

This menu is only available when you are logged in with a user with administrator rights.

A user with administrator rights can set permissions for each user group or define whether the related menu or action is available when no user is logged in.

Menus, options and other elements related to a permission may not be displayed or may be grayed out if a user has not the sufficient permissions. You can use the user group "common" as a collective group when you want to assign rights to every user group. For an easier understanding, each permission is described on the right.

 

Please be careful when assigning permissions to user groups. Some permissions are required by other permissions to work properly.

The permissions to see views of any type is e.g. required when the permission to edit a view of any type is granted.

The permissions svobserver.file.read and svobserver.file.write are necessary for saving the configuration to SVObserver.

Setting permission for a user group

  1. Scroll to the permission for which you want to configure the user group. You can find a description of each permission on the right.
  2. Tick or untick the checkbox under the user group.
  3. Click on "Save" in the top bar to save your changes in the WebApp configuration.
  4. If you are finished, click on the button "Back to WebApp" to go back to the WebApp.
  5. If you log in with a user for who you changed the permissions, menus, options and other elements may not be displayed or may be grayed out.
